# Chapter-18: CSV, JSON, and XML Files

## Introduction
This Chapter mainly working with structured data stored in text formats like CSV, JSON, and XML. I learned how programs store, transfer, and reuse data using serialization formats, which are widely used in real-world applications.
I also learned how to create, read, and manage CSV, JSON, and XML files directly using Python in the terminal.

## Explanation
This chapter explains how CSV, JSON, and XML files store data in different formats and how Python provides built-in modules to work with them easily. CSV is simple and works well for tabular data, JSON is flexible and widely used for APIs, and XML is more complex but still used in enterprise systems. 
this chapter explains how to read and write CSV files using the csv module and how JSON data can be converted into Python dictionaries and lists. The chapter explains why using Python modules is better than manual string handling when working with structured files

## Implementation
In this chapter, I created a Python program to write and read data from a CSV file using the csv module. First, the program creates a CSV file and writes sample data such as names and ages into it. After that, the same file is opened again in read mode to display all stored rows.

This implementation helped me understand how structured data is stored and accessed in real-world applications. I learned how CSV files are useful for handling tabular data and how Python simplifies file operations using built-in modules.

## Observations
* Structured data formats are important in real-world projects.
* CSV works best for spreadsheets and tabular data.
* JSON is simple and commonly used in APIs.
* Python modules make file handling easier and secure to use.
